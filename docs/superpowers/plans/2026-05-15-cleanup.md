### Task C6: Merge FATAL guards in run() and remove double-clear

**Files:**
- Modify: `src/supervisor/state_machine.cpp`

- [ ] **Step C6.1: Merge the two FATAL guard blocks**

In `src/supervisor/state_machine.cpp`, the current code has two `if (observedState_ != SystemState::FATAL)` blocks. Merge them into one since `consumeErrorEvent`/`consumeStateRequest` never change `observedState_`:

**Before** — two separate `if` guards testing the same condition. `consumeErrorEvent()` and `consumeStateRequest()` never change `observedState_`, so the second check always evaluates identically to the first:

```cpp
    ulTaskNotifyTake(pdTRUE, portMAX_DELAY);

    if (observedState_ != SystemState::FATAL) {           // check 1: drain events
        consumeErrorEvent();
        consumeStateRequest();      // never changes observedState_
    }

    if (observedState_ != SystemState::FATAL) {           // check 2: always same as 1
        if (targetState_ != observedState_ && !hasActiveOrchestration_) {
            stepTowardTarget();
        } else if (hasActiveOrchestration_) {
            checkOrchestrationResponse();
        } else if (observedState_ == SystemState::ERROR) {
            SystemState recoveryTarget = determineRecoveryTarget();
            if (recoveryTarget != observedState_) {
                postStateRequest(recoveryTarget);
            }
        }
    }
```

**After** — event processing and state stepping merged under a single FATAL guard:

```cpp
    if (observedState_ != SystemState::FATAL) {
        consumeErrorEvent();         // Phase 2: drain events
        consumeStateRequest();       // does NOT change observedState_

        // Phase 3: state stepping — same guard still holds (observedState_ unchanged above)
        if (targetState_ != observedState_ && !hasActiveOrchestration_) {
            stepTowardTarget();
        } else if (hasActiveOrchestration_) {
            checkOrchestrationResponse();
        } else if (observedState_ == SystemState::ERROR) {
            SystemState recoveryTarget = determineRecoveryTarget();
            if (recoveryTarget != observedState_) {
                postStateRequest(recoveryTarget);
            }
        }
    }
```

- [ ] **Step C6.2: Remove double-clear of hasActiveOrchestration_**

`hasActiveOrchestration_` is cleared in two places for the same event:

1. **`checkOrchestrationResponse()`** (orchestrator.cpp) — clears it at line 112 when a COMPLETED or TIMED_OUT response arrives. Then calls `setObservedState()` on COMPLETED.

2. **`setObservedState()`** (state_machine.cpp) — clears it again at line 208, even though `checkOrchestrationResponse()` already did.

Since `setObservedState()` is called from `checkOrchestrationResponse()` on COMPLETED, the second clear is a no-op write — the flag is already `false`. Remove it.

**Before:**
```cpp
void SupervisorV2::setObservedState(SystemState state) {
    PROD_LOG(kLogSource, "%s -> %s",
             stateToString(observedState_), stateToString(state));
    observedState_ = state;
    hasActiveOrchestration_ = false;  // ← already cleared by checkOrchestrationResponse()

    resetRecoveryIfOutOfError();
}
```

**After:**
```cpp
void SupervisorV2::setObservedState(SystemState state) {
    PROD_LOG(kLogSource, "%s -> %s",
             stateToString(observedState_), stateToString(state));
    observedState_ = state;
    // hasActiveOrchestration_ already cleared by checkOrchestrationResponse()
    resetRecoveryIfOutOfError();
}
```

### Task C7: Merge dual loops in startOrchestration()

**Files:**
- Modify: `src/supervisor/orchestrator.cpp`

- [ ] **Step C7.1: Merge the two component loops**

`startOrchestration()` iterates over `componentCount` twice — once to build the expected-bits mask (checking `!= DEGRADED`), once to call `postNextComponentState()` (checking `!= nullptr`). Both test `componentMailboxes_[i] == nullptr` and can be collapsed into a single pass.

**Before** — two loops over the same range:
```cpp
    // Loop 1: build expected-bits mask — only non-degraded registered components
    EventBits_t bits = 0;
    for (size_t i = 0; i < componentCount; i++) {
        if (componentMailboxes_[i] != nullptr
            && componentStatuses_[i] != ComponentStatus::DEGRADED) {
            bits |= (1 << i);
        }
    }

    xEventGroupClearBits(eventGroup_, kAllComponentBits);
    nextState_.transitionTarget = target;

    // Loop 2: write every registered component's mailbox
    for (size_t i = 0; i < componentCount; i++) {
        if (componentMailboxes_[i] != nullptr) {
            postNextComponentState(static_cast<ComponentID>(i));
        }
    }
```

**After** — single pass, early `continue` skips unregistered slots:
```cpp
    // Build expected-bits mask and write mailboxes in a single pass.
    // Unregistered components (nullptr) are skipped entirely.
    // DEGRADED components get a mailbox write but no event-group bit.
    EventBits_t bits = 0;
    for (size_t i = 0; i < componentCount; i++) {
        if (componentMailboxes_[i] == nullptr) continue;  // unregistered

        if (componentStatuses_[i] != ComponentStatus::DEGRADED) {
            bits |= (1 << i);                              // part of quorum
        }

        postNextComponentState(static_cast<ComponentID>(i)); // always write mailbox
    }
```
