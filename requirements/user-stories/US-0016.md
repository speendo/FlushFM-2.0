## User Story

As a **system orchestrator**,  
I want **to call component state methods and wait for completions, respecting required/optional policies**,  
so that **the system handles partial failures gracefully without blocking or crashing**.

---

## Acceptance Criteria

- [ ] SystemController internal state tracks: {currentTransition, targetState, componentsWaitingForCompletion}
- [ ] Define a static required/optional classification per component (component-level, not per target state)
- [ ] Transition flow implemented: call `component.setXXX(transitionId)` â†’ store returned timeoutMs â†’ wait for completion callbacks â†’ commit or revert
- [ ] Completion callbacks are mandatory for all components after `setXXX()`
- [ ] Handle callback `Completed`: mark component done
- [ ] Handle callback `Failed`: apply required/optional policy
- [ ] Policy for Required+Failed: do NOT commit target state; revert or enter ERROR state with logging
- [ ] Policy for Optional+Failed: mark as degraded but continue; commit target state anyway
- [ ] Use timeout returned by each `setXXX()` call as the watchdog value
- [ ] Enforce timeout: if a component does not report completion before timeout, SystemController calls `component.onTransitionTimeout(transitionId)`
- [ ] Timeout hook behavior: component enters a safe internal state and then reports `Failed` via callback (optional reason)
- [ ] Each transition gets a unique transitionId for logging and matching completions
- [ ] SystemController supports one in-flight transition per component; stale callbacks (old transitionId) are ignored
- [ ] On timeout, SystemController does not start a new transition for that component until timeout recovery has completed
- [ ] Log each state transition and result with transitionId for tracing in debug mode
- [ ] Log failed and timeouts in debug mode and production mode
- [ ] Test: manual transition test via serial CLI (e.g., send "transition idle")
- [ ] All existing state transitions (OFFâ†’STARTINGâ†’IDLEâ†’STREAMINGâ†’ERROR) work with new orchestration

## Notes

Required/Optional classification can be hardcoded for now in a static controller-owned config.
Example: REQUIRED = {WiFi, AudioRuntime}; OPTIONAL = {CLI, BoardInfo}
Timeout per component is returned by `setXXX(transitionId)` and treated as watchdog value.
On timeout, recovery is component-owned via `onTransitionTimeout(transitionId)` and completion callback.
No global timeout fallback; components MUST return valid timeout values for all target states.
