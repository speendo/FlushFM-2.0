# US-0003: Core Pinning Verification

> **Status:** To Do  
> **Priority:** High  
> **Created:** 2026-03-01

---

## User Story

As a **developer**,  
I want **to verify that ESP32-audioI2S can be effectively pinned to Core 1 via custom task wrapper**,  
so that **I can implement the dual-core architecture with confidence that audio tasks won't interfere with Core 0 system tasks**.

---

## Acceptance Criteria

*Each criterion should be testable and unambiguous. A story is "done" when all criteria are met.*

- [ ] Custom `audioEngineTask` successfully created and pinned to Core 1 using `xTaskCreatePinnedToCore`
- [ ] Audio streaming works correctly when `audio.loop()` is called from the Core 1 task
- [ ] Task monitoring confirms the main audio task runs exclusively on Core 1
- [ ] Task monitoring shows no unexpected audio-related tasks created on Core 0
- [ ] Performance comparison: custom wrapper vs library default (CPU usage, audio quality)
- [ ] Audio playback remains stable under simulated Core 0 load (busy loop test)
- [ ] Core 1 task can be suspended/resumed without audio corruption
- [ ] Memory usage patterns are acceptable (no excessive stack/heap consumption)
