# US-0004b: Component Lifecycle Integration Contract

> **Status:** To Do  
> **Priority:** High  
> **Created:** 2026-03-28

---

## User Story

As a **developer**,  
I want **each component to integrate through a clear lifecycle contract**,  
so that **setup and runtime responsibilities are consistent across modules**.

---

## Acceptance Criteria

- [ ] A common component lifecycle contract is introduced for orchestration-level integration
- [ ] Every component participating in orchestration exposes setup behavior through the contract
- [ ] Loop behavior is supported but optional for event-driven components
- [ ] `main.cpp` integrates components through the lifecycle contract instead of ad-hoc calls
- [ ] Event-driven modules are not forced into artificial polling loops
- [ ] Component dependencies are injected and not created internally
- [ ] No direct component-to-component state mutation is introduced

---

## Notes

- `setup` is mandatory for integrated components.
- `loop` is optional by design to keep WiFi and CLI efficient.
- Suggested implementation sequence:
	1. Define a minimal lifecycle contract interface (`setup`, optional `loop`).
	2. Adapt existing components to the contract without changing runtime behavior.
	3. Update orchestration to use contract-based registration and dispatch.
