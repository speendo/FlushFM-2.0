# US-0004e: Decouple CLI Output from Command Logic

> **Status:** Done  
> **Priority:** Medium  
> **Created:** 2026-03-28

---

## User Story

As a **developer**,  
I want **CLI command handling and CLI output rendering to be separated**,  
so that **command code stays readable, testable, and easier to maintain**.

---

## Acceptance Criteria

- [x] CLI command execution no longer writes output text directly inside command decision branches
- [x] A dedicated output layer/module is introduced for CLI messages (info, error, help, command result)
- [x] Command handlers return structured results (status + optional payload/message key) instead of printing directly
- [x] `cli.cpp` keeps orchestration responsibilities (read input, dispatch command, pass result to output layer)
- [x] Existing command behavior and user-visible CLI output remain functionally equivalent
- [x] Build succeeds in debug and production environments
- [x] At least one representative command path is covered by a unit or component-level test without relying on `Serial`

---

## Notes

- Keep migration incremental: start with helper output functions, then extract renderer and result model.
- Align log severity routing with existing debug guideline policy.
- Verified with successful `platformio run --environment debug` build.
- Verified with successful `platformio test --environment native` for `test_cli_command_logic`.
- Suggested implementation sequence:
	1. Introduce command-result structs and route one command path through them.
	2. Extract CLI output rendering into a dedicated module.
	3. Migrate remaining commands and add one test path without `Serial` dependency.
