# US-0025h: Arbitration and Mapping Regression Tests

> **Status:** Done | **Priority:** High | **Created:** 2026-04-25

## User Story
As a **maintainer**, I want **to lock-in behavioral safety for Mailbox ordering, stale/duplicate completion, and mapping outcomes**, so that **the event-handling safety properties are regression-protected**.

## Acceptance Criteria
- [x] Per-state/per-event normalized outcome matrix is documented and test-backed
  - For each `SystemState` and accepted `SystemEvent`, document the normalized outcome
- [x] Per-component signal-to-event mapping table is documented in code or comments
- [x] Single-slot Mailbox last-write-wins behavior is test-backed
- [x] Transition sub-state progression (`PENDING` -> `COMMITTED`/`FAILED`) is test-backed
- [x] Stale transition outcomes are ignored deterministically
- [x] Duplicate completion reports do not double-advance orchestration
- [x] Optional-component failures are reflected as `DEGRADED` and excluded from the quorum

## Notes
This story completes US-0025 event refactor work. It focuses on testing and behavioral documentation; no new feature work.
