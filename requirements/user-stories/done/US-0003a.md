# US-0003a: Code Cleanup & main.cpp Refactoring

> **Status:** Done  
> **Priority:** Medium  
> **Created:** 2026-03-11

---

## User Story

As a **developer**,  
I want **`main.cpp` to be slim and readable, with debug-only code cleanly separated**,  
so that **the production code path is easy to follow and debug tooling does not clutter the main file**.

---

## Acceptance Criteria

- [x] Serial CLI command dispatch is extracted into a dedicated `src/cli.h` / `src/cli.cpp` – `main.cpp` calls a single `cli::process(line)` function
- [x] WiFi helpers (`connectWiFi`, `onWiFiDisconnect`, `onWiFiConnect`) are extracted into `src/wifi_manager.h` / `src/wifi_manager.cpp`
- [x] Debug-only commands (`tasks`, `loadtest`, `suspend`, `resume`) live in `src/debug_cli.h` / `src/debug_cli.cpp`, guarded by `#ifdef DEBUG_ENABLED` at the **file level** – no `#ifdef` blocks scattered inside `cli.cpp` or `main.cpp`
- [x] `main.cpp` contains only: global wiring (object construction), `setup()`, `loop()`, and the audio task function – no inline command handling logic
- [x] All extracted modules follow the existing code style and are covered by a brief module-level comment

---

## Notes

**Refactoring target structure:**
```
src/
  main.cpp          ← wiring + setup/loop + audioTask only
  config.h          ← unchanged
  debug.h           ← unchanged
  cli.h / cli.cpp   ← Serial command dispatch (production commands)
  debug_cli.h / debug_cli.cpp  ← debug-only commands (tasks, loadtest, suspend, resume)
  wifi_manager.h / wifi_manager.cpp  ← WiFi connect + event handlers
```

**Debug CLI integration in `cli.cpp`:**
```cpp
#ifdef DEBUG_ENABLED
#include "debug_cli.h"
// register debug commands
#endif
```
This keeps a single include guard at the top of the file rather than scattered `#ifdef` blocks throughout.
