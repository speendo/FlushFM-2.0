# US-0006: Persistent Settings via NVS

> **Status:** Done  
> **Priority:** High  
> **Created:** 2026-03-11

---

## User Story

As a **developer**,  
I want **WiFi credentials and the last-played station URL to be stored in NVS and restored automatically on boot**,  
so that **I don't need to re-enter them after every reflash or reboot**.

---

## Acceptance Criteria

- [x] WiFi SSID and password are saved to NVS when set via the `ssid` / `pass` Serial commands
- [x] Station URL is saved to NVS when a stream is started via `play` or `switch`
- [x] On boot, credentials and station URL are restored from NVS automatically
- [x] If credentials are present on boot, WiFi connects automatically without manual `connect`
- [x] If a station URL is stored and boot auto-connect succeeds, playback starts automatically
- [x] A `play` command without URL argument uses the last-saved station from NVS
- [x] A `forget` Serial command clears all NVS settings and prevents auto-connect
- [x] A `reset` Serial command resets only the runtime session (stop stream + disconnect WiFi + clear volatile credentials) without deleting NVS settings
- [x] NVS read/write is encapsulated in a dedicated `Settings` module (`lib/settings/`)
- [x] NVS survives a firmware reflash (no `erase` target); verified manually
- [x] Never store `ssid`, `pass` or `station` in the flash storage, only use NVS

---

## Notes

- Use `Preferences` (Arduino-ESP32 wrapper around NVS) – approved in `software-architecture.md`
- NVS namespace: `flushfm`
- Keys: `ssid`, `pass`, `station`
- Credentials must not be logged in plaintext (password masked in log output)
- `Settings` module depends only on `Preferences` – no dependency on audio or WiFi layers
- Wiring into `main.cpp` (auto-connect, auto-play) is orchestration, not part of the library
- Auto-play is boot-path only and must not trigger on later manual `connect` or reconnect events
