## User Story

As a **developer**,  
I want **to set up the basic PlatformIO ESP32 project structure with minimal boilerplate code**,  
so that **I have a clean foundation for implementing and testing FlushFM features**.

## Notes

*This is the foundation step before all other user stories. Keep it minimal but functional.*

**Required PlatformIO project structure:**
```
FlushFM 2.0/
├── platformio.ini         # PlatformIO configuration
├── src/                   # Main source code
│   ├── main.cpp          # Entry point (setup() and loop())
│   └── config.h          # Build-time configuration constants
├── lib/                  # Project-specific libraries (empty for now)
├── test/                 # Unit tests (empty for now)
├── data/                 # LittleFS content (empty for now)
└── boards/               # Custom board definitions (if needed)
```

**Minimal boilerplate requirements:**
- Serial initialization (115200 baud)
- WiFi scan functionality (no connection required yet)
- Basic error handling (try/catch or status checks)
- Board identification (confirm target hardware detection)
- Memory info display (Flash, available RAM/PSRAM - see hardware.md)
- PlatformIO environment configuration for production and debug builds

**Libraries to include:**
- Arduino-ESP32 framework (via PlatformIO)
- WiFi library (built-in)
- No external libraries yet - keep it minimal

**PlatformIO requirements:**
- Target hardware board configuration (see hardware.md)
- Debug and production build environments
- Proper library dependency management
- Monitor speed configuration (115200 baud)
