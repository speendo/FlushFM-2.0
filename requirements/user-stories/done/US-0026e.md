# US-0026e: Completion Flow Migration (reportCompletion + in-flight invariants)

> **Status:** Done | **Priority:** Medium | **Created:** 2026-04-25

## User Story
As a **maintainer**, I want **to migrate completion reporting and in-flight orchestration invariants**, so that **the completion flow uses static identifiers and respects orchestration ownership**.

## Acceptance Criteria
- [x] Completion reporting uses static identifiers in controller logic
- [x] In-flight entries are constrained to active orchestration transition context
- [x] Legacy public transition helpers are removed or made internal-only if still needed

## Notes
This story depends on US-0026d and focuses on the completion callback and in-flight transition tracking.

### Implementation
- `pendingTransitions_` and `componentHooks_` migrated from `std::map<std::string, ...>` to `std::array<..., Count>`
- `#include <map>` removed from supervisor.h (no remaining map users)
- `setComponentTransitionHooks(ComponentID, ...)` overload added — direct array access
- `reportCompletion(ComponentID, ...)` overload added — array-based pending + registry access
- `beginComponentTransition(ComponentID, ...)` overload added — stays public for test compatibility
- Both `const char*` overloads delegate to their `ComponentID` counterparts via `componentIdFromName()`
- `beginOrchestration`: pending transition reset uses `for (auto& p : pendingTransitions_) p = {};`, iteration uses array indices with `return componentName(static_cast<ComponentID>(i))` for log labels
- `checkTransitionTimeouts`: iteration uses array indices, hooks accessed via `componentHooks_[i]`
- `componentsWaitingForCompletion`: counts array entries with `transitionId != 0`
- `test_transition_completion_tracking/test_main.cpp`: fixed pre-existing `SystemController` → `Supervisor` and migrated all call sites to `ComponentID` overloads; now passes (6 tests)
