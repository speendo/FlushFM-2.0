# US-0026f: Runtime Ballast Removal (normalize/reason-copy/name-guards)

> **Status:** Done | **Priority:** Medium | **Created:** 2026-04-25

## User Story
As a **maintainer**, I want **to remove string normalization and mutable reason-copy processing from hot paths**, so that **the controller remains lightweight for ESP32 without changing behavior**.

## Acceptance Criteria
- [x] Removed helpers are absent from production controller paths
- [x] Failure control-path logic is independent from free-form text
- [x] Diagnostics use stable static semantics

## Notes
This story depends on US-0026e and completes the US-0026 static-identifier refactor work.

### Implementation
- `ComponentRegistryEntry::lastFailureReason` changed from `char[513]` to `const char*` (no copy, static lifetime)
- `kMaxFailureReasonLen` removed from `component_types.h`
- `copyFailureReason()` removed from `supervisor.h` declaration and `supervisor.cpp` body
- 6 call sites replaced with direct pointer assignment (`= nullptr` or `= reason`)
- Build and 55 tests pass; no regressions
