# US-0030: Rename EventPolicy Values to BestEffort and Critical

> **Status:** Done | **Priority:** Medium | **Created:** 2026-05-08

## User Story
As a **maintainer**, I want **EventPolicy enum values renamed to clearer semantics**, so that **the intended post-event handling behavior is immediately obvious from the name**.

## Acceptance Criteria
- [x] `EventPolicy::FIRE_AND_FORGET` is renamed to `EventPolicy::BestEffort`
- [x] `EventPolicy::BOUNDED_BLOCKING` is renamed to `EventPolicy::Critical`
- [x] All references in header files (supervisor.h) are updated
- [x] All implementation references in supervisor.cpp are updated
- [x] All test files using EventPolicy values are updated
- [x] All component files and main.cpp using EventPolicy are updated
- [x] Documentation of EventPolicy semantics is updated (comments/guidelines)
- [x] Debug and production builds succeed without errors
- [x] All native unit tests pass (`pio test -e native`)

## Notes
- `BestEffort` indicates the event may be dropped if the queue is full (non-blocking)
- `Critical` indicates the event is important and should wait briefly if needed (bounded-blocking)
- This aligns with US-0025c which marked this as a decided naming convention
- No functional behavior changes; this is a naming clarification only
- 4/10 test suites pass cleanly; 6 have pre-existing compilation failures (unrelated `SystemController` references in test files from pre-US-0029 state). All 4 suites that exercise EventPolicy logic pass.
- Affected files: supervisor.h, supervisor.cpp, test_component_types/test_main.cpp, main.cpp, system_components.cpp, debug_cli.cpp, cli.cpp
