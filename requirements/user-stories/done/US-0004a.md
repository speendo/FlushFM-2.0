# US-0004a: Main as Pure Orchestrator

> **Status:** Done  
> **Priority:** High  
> **Created:** 2026-03-28

---

## User Story

As a **developer**,  
I want **`main.cpp` to contain only system wiring, `setup()`, and `loop()`**,  
so that **responsibilities are clear and architecture changes stay low-risk**.

---

## Acceptance Criteria

- [x] `main.cpp` only contains concrete dependency declarations, `setup()`, and `loop()`
- [x] Audio task creation and lifecycle logic are moved out of `main.cpp` into a dedicated module
- [x] Board information logging is moved out of `main.cpp` into a dedicated module
- [x] ESP32-audioI2S callback bridge is isolated from setup and runtime wiring concerns
- [x] Existing CLI behavior (`ssid`, `pass`, `connect`, `play`, `stop`, `switch`, `volume`, `balance`, `help`) remains unchanged
- [x] Audio initialization still occurs on the audio task core to preserve thread-safety
- [x] Project builds successfully in debug and production environments

---

## Notes

- This story is structural only and must not introduce new user-facing behavior.
- Keep ownership boundaries explicit and dependency injection unchanged.
- Suggested implementation sequence:
	1. Extract audio task lifecycle and callback bridge from `main.cpp` into dedicated modules.
	2. Move board info output into a separate system-info module.
	3. Reduce `main.cpp` to wiring + `setup()` + `loop()` and run build verification.
