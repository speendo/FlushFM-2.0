# US-0036: Event System Consolidation — Remove Legacy Events and Transition Queue

> **Status:** Done | **Priority:** Medium | **Created:** 2026-05-09 | **Resolved:** 2026-05-17

## User Story
As a **maintainer**, I want **all state requests to use a single generic `STATE_REQUESTED` event type with no legacy-specific events and no transition queuing**, so that **the event system aligns with `state-management.md`'s single-slot Mailbox model and the no-queue policy**.

## Acceptance Criteria
- [x] `PLAY_REQUESTED`, `STOP_REQUESTED`, `ENTER_SLEEP`, `RECOVER` removed from `SystemEvent` enum
- [x] Only `COMPONENT_SETUP_FAILED` and `STATE_REQUESTED` remain in the enum
- [x] All callers (CLI, debug CLI, main.cpp, tests) migrated to `STATE_REQUESTED(targetState)` via 3-param `postEvent`
- [x] Internal `handleEvent()` auto-play calls use `STATE_REQUESTED` with `mailbox_.targetState` set first
- [x] `handleEvent()` has no duplicated legacy if-chain and no empty `switch (observedState_)`
- [x] `TransitionRequestDecision` enum removed entirely; `requestTransition()` returns `bool`
- [x] `hasQueuedStateTransition_` / `queuedStateTransition_` removed from Supervisor
- [x] All public queued-transition accessors (`hasQueuedTransition()`, `queuedTransitionId()`, etc.) removed
- [x] `docs/event-contract-model.md` updated to reflect the 2-event enum
- [x] Debug and production builds succeed without errors
- [x] All native unit tests pass

## Resolution Notes

This story's goals were inherently satisfied by the SupervisorV2 architecture used in the production build (`PRODUCTION_BUILD`).

**SupervisorV2 (`src/supervisor/supervisor_v2.h/cpp`)** was built after the legacy event cleanup (US-0032, US-0035) and never had:
- `PLAY_REQUESTED`, `STOP_REQUESTED`, `ENTER_SLEEP`, `RECOVER` — uses `postStateRequest(target)` instead
- `TransitionRequestDecision` enum — `postStateRequest` returns `void`
- `hasQueuedStateTransition_` / `queuedStateTransition_` — single-slot mailbox, no queue
- Per-state specific event handlers — `run()` uses a simple phase pipeline (drain events → step → check orchestration)

The **legacy Supervisor** (`src/supervisor/supervisor.h/cpp`, compiled only in `native` env) was separately cleaned up by US-0032 and US-0035, reducing its `SystemEvent` enum to just `{COMPONENT_SETUP_FAILED, STATE_REQUESTED}` with no transition queue. Its `handleEvent()` has a clean 2-branch dispatch with no legacy if-chain or empty switch.

The production event model is:
- **State requests**: `postStateRequest(targetState)` — writes single-slot Mailbox (last-write-wins)
- **Error signals**: `postErrorEvent(reason, source)` — first-writer-wins ErrorEvent

This aligns with `state-management.md`'s single-slot Mailbox model and no-queue policy.
