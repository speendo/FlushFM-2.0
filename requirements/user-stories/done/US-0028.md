# US-0028: Target State Tracking for Multi-Step Transitions

> **Status:** Done | **Priority:** Medium | **Created:** 2026-05-02

## User Story
As a **maintainer**, I want **the controller to track a target state that persists across intermediate transitions**, so that **PLAY_REQUESTED from SLEEP reliably reaches LIVE without relying on deferred event flags**.

## Problem Description
Currently, when `PLAY_REQUESTED` is posted while in `SLEEP` state:
1. `pendingPlayAfterReady_` flag is set
2. State transitions to `CONNECTING`
3. When `AUDIO_INIT_OK` + `WIFI_READY` arrive, transition to `READY` completes
4. `deferredPlayAfterReadyEvent_` flag triggers `handleEvent(PLAY_REQUESTED)` to start `READY → LIVE`
5. This flag-based approach is fragile and causes test failures (test_state_transition_flow.cpp:314)

**Root cause:** The controller lacks a persistent `targetState_` variable. Instead, it uses temporary flags (`pendingPlayAfterReady_`, `deferredPlayAfterReadyEvent_`) that are hard to trace and debug.

## Acceptance Criteria
- [x] Add `SystemState targetState_` to SystemController (initialized to `SLEEP`)
- [x] When `PLAY_REQUESTED` is handled, set `targetState_ = LIVE`
- [x] When `CONNECTING → READY` transition completes, check `if (targetState_ == LIVE)` and auto-trigger `READY → LIVE`
- [x] Remove `pendingPlayAfterReady_` and `deferredPlayAfterReadyEvent_` flags
- [x] Update `transitionTo()` to clear `targetState_` when reaching the target
- [x] All existing tests pass (test_state_transition_flow.cpp:314 should now pass)
