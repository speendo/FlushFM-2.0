# US-0026: Static Component Identity Refactor (Umbrella)

> **Status:** To Do | **Priority:** Medium | **Created:** 2026-04-19

## User Story
As a **maintainer**, I want **the component-identity refactor to be split into a functional migration and a lightweight cleanup phase**, so that **scope stays clear and risk stays controlled**.

## Acceptance Criteria
- [ ] `US-0026a` defines and delivers the functional migration from name-based logic to static component identifiers
- [ ] `US-0026b` defines and delivers the runtime-ballast cleanup after the functional migration is in place
- [ ] No separate runtime behavior is introduced for test vs production; logic remains single-path and deterministic
- [ ] Graceful degradation behavior remains intact across both substories

## Notes
This is an umbrella story only. Implementation scope is split across `US-0026a` and `US-0026b`.
