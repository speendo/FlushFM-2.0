# US-0035: Remove Component Commands and BOOT Event from Supervisor

> **Status:** In Progress | **Priority:** High | **Created:** 2026-05-09

## User Story
As a **maintainer**, I want **component commands (WIFI_READY, AUDIO_INIT_OK, etc.) and the pseudo BOOT event removed from the Supervisor**, so that **the Supervisor only handles global state transitions per state-management.md and supervisor.cpp is lean**.

## Acceptance Criteria
- [x] Error Event flag `{pending, reason, source}` added — set by components, checked in `processMailbox()` to trigger ERROR transitions
- [x] `setErrorEvent()` method on Supervisor accepts only the first error per transition (duplicates ignored)
- [x] Component command handlers (SLEEP, CONNECTING, READY, LIVE case blocks) removed from `handleEvent()`
- [x] PLAY_REQUESTED/SLEEP quick path (`getComponentStatus()` readiness check) removed — always orchestrate
- [x] `COMPONENT_SETUP_FAILED` is state-independent in `handleEvent()` — triggers ERROR from any state
- [x] `WIFI_READY`, `AUDIO_INIT_OK`, `AUDIO_INIT_FAILED`, `WIFI_DISCONNECTED`, `STREAM_LOST` removed from `SystemEvent` enum
- [x] WiFi and Audio components updated to use `setErrorEvent()` instead of `postEvent(WIFI_DISCONNECTED/AUDIO_INIT_FAILED)`
- [x] WiFi and Audio components stop posting WIFI_READY/AUDIO_INIT_OK — orchestration handles readiness via COMMITTED
- [x] `debug_cli.cpp` error trigger updated from `AUDIO_INIT_FAILED` to `COMPONENT_SETUP_FAILED`
- [ ] `BOOT` removed from `SystemEvent` enum (per guidelines: BOOT is not a System Event)
- [ ] `BOOTING` case block removed from `handleEvent()` (no events handled in BOOTING — transient state)
- [ ] Boot sequence: constructor sets `BOOTING`, `COMPONENT_SETUP_FAILED` handling replaces `BOOT` for error detection, no manual `postEvent(BOOT)` needed
- [ ] `ErrorEvent` double-fault: if `COMPONENT_SETUP_FAILED` fires while already in `ERROR`, transition to `FATAL`
- [ ] `test_component_types` SystemEvent round-trip updated (BOOT removed, 5 events remain)
- [ ] Canonical test set covers:
  - `SLEEP → READY`, `SLEEP → LIVE`, `READY → LIVE`
  - `LIVE → READY`, `LIVE → SLEEP`, `READY → SLEEP`
  - `ALL → ERROR` (via `COMPONENT_SETUP_FAILED` and via `ErrorEvent` flag)
  - `ERROR → FATAL` (double-fault, placeholder)
  - ObservedState lag semantics
  - Optional component failure
  - Matrix timeout enforcement
  - FATAL absorbing state
- [ ] All tests pass; debug and production builds succeed
