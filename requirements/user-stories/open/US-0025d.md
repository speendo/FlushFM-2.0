# US-0025d: Enum/String Single Source Mapping

> **Status:** To Do | **Priority:** High | **Created:** 2026-04-25

## User Story
As a **maintainer**, I want **to remove duplicated enum-label maintenance**, so that **enum names and string labels are derived from one central source**.

## Acceptance Criteria
- [ ] SystemEvent, policy enum, and reason labels derive from one source
- [ ] No duplicate switch-based label tables remain for migrated enums
- [ ] Build succeeds in debug and production

## Notes
This story depends on contract decisions from US-0025c and applies them to the shared enum-to-string mapping mechanism.
