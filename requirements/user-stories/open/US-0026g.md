# US-0026g: Remove String-Based Shims and Migrate Callers

> **Status:** To Do | **Priority:** Medium | **Created:** 2026-05-09

## User Story
As a **maintainer**, I want **all remaining string-based Supervisor API overloads removed from the public API and all callers migrated to ComponentID overloads**, so that **the ComponentID migration is complete and no production code depends on runtime string names**.

## Acceptance Criteria
- [ ] All production callers in `system_components.cpp`, `cli.cpp`, and `debug_cli.cpp` use `ComponentID` overloads instead of string-based overloads
- [ ] All passing test files (`test_state_transition_flow`, `test_transition_completion_tracking`) use `ComponentID` overloads throughout; string overloads are not called from test code
- [ ] 7 string-based public overloads removed from `Supervisor`: `registerComponent(const char*)`, `setComponentTransitionHooks(const char*)`, `getComponentStatus(const char*)`, `isComponentRequired(const char*)`, `markComponentFailed(const char*)`, `reportCompletion(const char*)`, `beginComponentTransition(const char*)`
- [ ] `normalizeComponentName()` removed from `supervisor.h` and `supervisor.cpp`
- [ ] `componentIdFromName()` removed from `supervisor.cpp`
- [ ] `#include <string>` removed from `supervisor.h`
- [ ] `kMaxComponentNameLen` removed from `component_types.h`
- [ ] `isIntentEvent()` and `deferredIntentEvents_` remain unchanged (scope of event model cleanup, not ComponentID)
- [ ] All previously passing tests remain passing after migration
- [ ] Debug and production builds succeed

## Notes
This story completes the US-0026 static-identifier refactor. It removes the string-based shims that were introduced in US-0026c and kept through US-0026d/e as transitional compatibility. After this story, the Supervisor API accepts only `ComponentID` — no `const char*` component names.

### Call sites to migrate (production)

**`system_components.cpp`** — 4 component classes (BoardInfo, WiFi, AudioRuntime, CLI):
- `registerComponent(name(), required)` → `registerComponent(id(), required)` (4 sites)
- `setComponentTransitionHooks(name(), ...)` → `(ComponentID, ...)` (4 sites + lambda body changes)
- `reportCompletion(name(), ...)` → `reportCompletion(id(), ...)` (4 sites)

**`cli.cpp`** — 2 sites:
- `getComponentStatus(name)` → `getComponentStatus(id)`

**`debug_cli.cpp`** — 2 sites:
- `getComponentStatus(name)` → `getComponentStatus(id)`
- `isComponentRequired(name)` → `isComponentRequired(id)`

### Call sites to migrate (tests)

**`test_state_transition_flow`** — TransitionHooksFixture:
- Component array: `const char*` → `ComponentID`
- `registerComponent("WiFi", ...)` → `registerComponent(ComponentID::WiFi, ...)` (4 sites)
- `setComponentTransitionHooks(name, ...)` → `(ComponentID, ...)` (4 sites in loop)
- Lambda captures: `void(controller.reportCompletion(name, ...)` → `(ComponentID, ...)`
- `completeAllActive`: string iteration → `ComponentID` iteration

**`test_transition_completion_tracking`** — 4 sites:
- `registerComponent("WiFi", ...)` → `registerComponent(ComponentID::WiFi, ...)`
- `registerComponent("AudioRuntime", ...)` → `registerComponent(ComponentID::AudioRuntime, ...)`
- `registerComponent("CLI", ...)` → `registerComponent(ComponentID::CLI, ...)`
- `registerComponent("BoardInfo", ...)` → `registerComponent(ComponentID::BoardInfo, ...)`

### What stays unchanged
- 4 broken test suites (already erroring on `SystemController` naming — not in scope)
- `isIntentEvent()` and deferred intent queue logic (event model cleanup — US-0025b scope)
- `copyFailureReason()` and `kMaxFailureReasonLen` (handled in US-0026f)
- `#include <vector>` (used by observers_ and deferredIntentEvents_)
