# US-0027a: SystemController Module Boundary Spec

> **Status:** To Do | **Priority:** Medium | **Created:** 2026-04-25

## User Story
As a **maintainer**, I want **to define SystemController as a module/orchestrator boundary in explicit terms**, so that **future refactors do not reintroduce controller-as-component coupling**.

## Acceptance Criteria
- [ ] SystemController is explicitly non-component in architecture contract
- [ ] Public API responsibilities are listed and bounded
- [ ] Internal methods (for example transition commit helper) are explicitly excluded from public surface

## Notes
This story is specification-only. No code refactoring; only boundary documentation and approval.
