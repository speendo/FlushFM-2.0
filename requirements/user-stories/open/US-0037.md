# US-0037: Algorithmic Step-Through for STATE_REQUESTED

> **Status:** To Do | **Priority:** Medium | **Created:** 2026-05-09

## User Story
As a **maintainer**, I want the `STATE_REQUESTED` handler to use an algorithmic step-through approach instead of a per-target switch, so that the state hierarchy (encoded in `stateRank`) alone drives transitions and the handler has zero per-state duplication.

## Acceptance Criteria
- [ ] The `STATE_REQUESTED` switch in `handleEvent()` is replaced with an algorithmic approach using `stateRank()`
- [ ] ERROR and FATAL targets cause direct `transitionTo()` — no orchestration
- [ ] SLEEP target from any L2 state steps backward through L2: `LIVE → READY → SLEEP` or `READY → SLEEP`
- [ ] L2 targets (BOOTING, CONNECTING, READY, LIVE) step toward the target one state at a time via `requestStateTransition()`
- [ ] `targetMode_` is set once as the ultimate goal; the step logic reads it to determine direction
- [ ] `reportCompletion()` auto-advances toward `targetMode_` when `observedState_ != targetMode_` (generalized from the current hardcoded READY→LIVE check)
- [ ] LIVE→LIVE replay pattern: sets `targetMode_ = LIVE`, steps to READY, auto-advances back to LIVE
- [ ] CONNECTING deferral: requesting LIVE during CONNECTING sets `targetMode_ = LIVE` and returns without cancelling the in-flight transition
- [ ] Transient states (BOOTING, CONNECTING) requested as targets are silently ignored
- [ ] FATAL guard at top of `handleEvent()` prevents any processing
- [ ] `COMPONENT_SETUP_FAILED` handler unchanged
- [ ] Debug and production builds succeed without errors
- [ ] All native unit tests pass

## Notes
- The state hierarchy is already encoded: `FATAL(0) < ERROR(10) < SLEEP(20) < BOOTING(30) < CONNECTING(40) < READY(50) < LIVE(60)` via `stateRank()`
- L2 linear sequence is `BOOTING(30) → CONNECTING(40) → READY(50) → LIVE(60)` — stepping is ±10 on rank
- SLEEP(20) is L1, not in L2 — going to SLEEP from any L2 state must step backward through L2 first
- `reportCompletion()` currently has a hardcoded check: `if (observedState_ == READY && targetMode_ == LIVE) → auto-play`. This generalizes to: `if (observedState_ != targetMode_) → requestTransition(nextStep(toward targetMode_))`
- The current switch (lines 441-485 of supervisor.cpp) encodes step-through logic per-case. This algorithm eliminates all case-by-case branching.
