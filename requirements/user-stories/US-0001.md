# US-0001: Basic Audio Connection Test

> **Status:** To Do  
> **Priority:** High  
> **Created:** 2026-03-01

---

## User Story

As a **developer**,  
I want **to verify that ESP32-audioI2S can successfully connect to WiFi and stream internet radio data**,  
so that **I can confirm the network streaming capability works independently of audio output hardware**.

---

## Acceptance Criteria

*Each criterion should be testable and unambiguous. A story is "done" when all criteria are met.*

- [ ] ESP32-S3 successfully connects to WiFi
- [ ] ESP32-audioI2S library initializes without errors
- [ ] Device successfully connects to internet radio station URLs
- [ ] Stream data is being received (confirmed via Serial monitor output)
- [ ] Library reports stream metadata (station name, track info, bitrate) if available
- [ ] Stream can be stopped and restarted without device reset
- [ ] Device handles brief network interruption gracefully (no crash, reconnection)
- [ ] Stream switching works (change from one station URL to another)

---

## Notes

*This test focuses purely on network streaming capability. No audio output hardware required - just confirm the ESP32 can fetch and decode stream data.*

**Test setup:**
- ESP32-S3-D-1 N16R8
- Stable WiFi network
- Serial monitor for debugging
- NO audio hardware needed yet

**Test stations (known working streams):**
- http://ice1.somafm.com/groovesalad-256-mp3 (SomaFM Groove Salad)
- http://stream.srg-ssr.ch/m/drs3/mp3_128 (SRF 3)

**Libraries to include:**
- Arduino-ESP32 framework
- ESP32-audioI2S (schreibfaul1)

**Success indicators:**
- Serial output shows "Connected to [station]" messages
- Stream metadata appears in Serial monitor
- No crash/reset when switching stations
- Library internal buffers are working (no buffer underrun errors)

**Expected I2S connections:**
- BCK (Bit Clock) → PCM5102A SCK
- WS (Word Select) → PCM5102A LCK  
- DIN (Data In) → PCM5102A DIN
