# US-0005: Persistent Settings via NVS

> **Status:** To Do  
> **Priority:** High  
> **Created:** 2026-03-11

---

## User Story

As a **developer**,  
I want **WiFi credentials and the last-played station URL to be stored in NVS and restored automatically on boot**,  
so that **I don't need to re-enter them after every reflash or reboot**.

---

## Acceptance Criteria

- [ ] WiFi SSID and password are saved to NVS when set via the `ssid` / `pass` Serial commands
- [ ] Station URL is saved to NVS when a stream is started via `play` or `switch`
- [ ] On boot, credentials and station URL are restored from NVS automatically
- [ ] If credentials are present on boot, WiFi connects automatically without manual `connect`
- [ ] If a station URL is stored and WiFi connects successfully, playback starts automatically
- [ ] A `forget` Serial command clears all NVS settings and prevents auto-connect
- [ ] NVS read/write is encapsulated in a dedicated `Settings` module (`lib/settings/`)
- [ ] NVS survives a firmware reflash (no `erase` target); verified manually
- [ ] Never store `ssid`, `pass` or `station` in the flash storage, only use NVS

---

## Notes

- Use `Preferences` (Arduino-ESP32 wrapper around NVS) – approved in `software-architecture.md`
- NVS namespace: `flushfm`
- Keys: `ssid`, `pass`, `station`
- Credentials must not be logged in plaintext (password masked in log output)
- `Settings` module depends only on `Preferences` – no dependency on audio or WiFi layers
- Wiring into `main.cpp` (auto-connect, auto-play) is orchestration, not part of the library
