# US-0002: Audio Output Chain

> **Status:** To Do  
> **Priority:** High  
> **Created:** 2026-03-01

---

## User Story

As a **developer**,  
I want **to verify that the I2S → DAC → speakers audio output chain works correctly with streamed data**,  
so that **I can confirm the complete audio path from network stream to audible sound**.

---

## Acceptance Criteria

*Each criterion should be testable and unambiguous. A story is "done" when all criteria are met.*

- [ ] I2S pinout is correctly configured for PCM5102A DAC (BCK, WS, DIN)
- [ ] PCM5102A DAC is properly wired and powered
- [ ] Audio output is audible and clear through connected speakers/headphones
- [ ] Volume control works (if supported by library)
- [ ] Audio quality is good (no distortion, crackling, or dropouts)
- [ ] Stereo channels work correctly (left/right separation)
- [ ] No audio artifacts during stream switching
- [ ] Audio stops cleanly when stream is disconnected

---

## Notes

*This test validates the physical audio output chain. Requires US-0001 (WiFi streaming) to be working first.*

**Test setup:**
- ESP32-S3-D-1 N16R8 (with working WiFi streaming from US-0001)
- PCM5102A DAC breakout board
- Breadboard connections for I2S
- Powered speakers or headphones
- Multimeter for voltage verification (optional)

**I2S connections to verify:**
- BCK (Bit Clock) → PCM5102A SCK
- WS (Word Select) → PCM5102A LCK  
- DIN (Data In) → PCM5102A DIN
- Power: 3.3V/5V and GND

**Test procedure:**
1. Use working stream connection from US-0001
2. Add I2S hardware and configuration
3. Verify audio output with known good stream
4. Test volume control and audio quality

**Troubleshooting indicators:**
- No audio: Check I2S wiring, DAC power, speaker connections
- Distorted audio: Check sample rate, bit depth, I2S clock timing
- One channel missing: Check stereo wiring, WS signal
