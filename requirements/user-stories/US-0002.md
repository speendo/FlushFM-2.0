# US-0002: Audio Output Chain

> **Status:** Done  
> **Priority:** High  
> **Created:** 2026-03-01

---

## User Story

As a **developer**,  
I want **to verify that the I2S → DAC → speakers audio output chain works correctly with streamed data**,  
so that **I can confirm the complete audio path from network stream to audible sound**.

---

## Acceptance Criteria

*Each criterion should be testable and unambiguous. A story is "done" when all criteria are met.*

- [x] I2S pinout is correctly configured for PCM5102A DAC (BCK, WS, DIN)
- [x] PCM5102A DAC is properly wired and powered
- [x] Audio output is audible and clear through connected speakers/headphones
- [x] Volume control works (if supported by library)
- [x] Audio quality is good (no distortion, crackling, or dropouts)
- [x] Stereo channels work correctly (left/right separation)
- [x] No audio artifacts during stream switching
- [x] Audio stops cleanly when stream is disconnected

---

## Notes

Requires US-0001 completed. I2S pinout and PCM5102A wiring → `pinout.md`, `hardware.md`.

**Test procedure:**
1. Use working stream from US-0001
2. Add I2S hardware and verify audio output with known good stream
3. Test volume control and stereo separation
