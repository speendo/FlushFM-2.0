# US-0010: Reduce Audio.h Verbose Logging

> **Status:** To Do  
> **Priority:** Low  
> **Created:** 2026-03-28

---

## User Story

As a **developer**,  
I want **debug output from Audio.h to be less verbose in debug builds**,  
so that **meaningful application logs are not drowned out by audio library internals**.

---

## Acceptance Criteria

- [ ] Audio library debug output (e.g., `AUDIO_LOG_IMPL()` messages) can be suppressed via build flag or runtime setting
- [ ] Application debug logs (`PROD_LOG`, `ERROR_LOG`, `DEBUG_LOG`) remain visible and readable
- [ ] Verbose audio logging can be re-enabled if needed for audio-specific debugging
- [ ] Solution does not require modifying ESP32-audioI2S library source directly
- [ ] Debug build without suppression still works as before (no regression)

---

## Investigation Notes

- **Primary Issue:** Audio.h produces continuous output like:
  ```
  [ 89752][D][Audio.h:640] AUDIO_LOG_IMPL(): Audio.cpp:4753 m_audioDataReadPtr 480384, m_audioDataSize 0
  ```
  - **Reproduced With:** https://orf-live.ors-shoutcast.at/fm4-q2a (ORF FM4 station)
  - **Library:** `ESP32-audioI2S` (version 3.4.4)

- **Secondary Issue (Regression from US-0004a-f):** Application-level DEBUG_LOG statements too verbose
  - **Symptom:** `system_controller.cpp` logs "Dispatching event: ..." on every event (high frequency during streaming)
  - **Root Cause:** Violates `debug.md` rule → "Log state changes, not continuous operations"
  - **Impact:** Output unreadable even in debug mode; makes `monitor` output difficult to parse
  - **Fix:** Remove per-event dispatch logs; keep only state-transition logs (already using `PROD_LOG`)

- **Overall Impact:** Makes `monitor` output difficult to read and obscures meaningful application logs
- **Approach Options:**
  1. Configure `CORE_DEBUG_LEVEL` in `platformio.ini` to exclude Audio.h logs
  2. Add conditional wrapper in `AudioPlayerESP32` to suppress internal library output
  3. Redirect Audio.h logs to separate handler (conditional compilation)
  4. Audit `system_controller.cpp` and other components for over-logging violation of debug.md
