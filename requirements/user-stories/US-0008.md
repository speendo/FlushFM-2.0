# US-0008: LED State Visualization

> **Status:** To Do  
> **Priority:** Medium  
> **Created:** 2026-03-28

---

## User Story

As a **user**,  
I want **the onboard LED to visually reflect the system's main operational state**,  
so that **I can quickly see FlushFM's status at a glance without accessing the display or CLI**.

---

## Acceptance Criteria

- [ ] LED controller component implemented and integrated into `SystemComponents` lifecycle
- [ ] LED indication maps all five `SystemState` values to distinct visual states (OFF, STARTING, IDLE, STREAMING, ERROR)
- [ ] LED state updates whenever `SystemController` transitions to a new main state
- [ ] Color and brightness mapping is configurable per state (at minimum via compile-time config table, optionally via CLI)
- [ ] IDLE indication is configurable: fully off or very low brightness (strong dimming)
- [ ] Color/pattern default profile is documented and intuitive (e.g., GREEN=IDLE, BLUE=STREAMING, RED=ERROR, ORANGE=STARTING, DARK=OFF)
- [ ] PWM brightness control implemented
- [ ] LED initialization succeeds on boot (GPIO configured, no blocking delays)
- [ ] Debug and production builds both report LED setup success in logs

---

## Design Notes

### Hardware Assumption
- **Target Board:** YD-ESP32-2.3
- **LED Type:** Onboard RGB status LED available on board variant (exact controller/pin mapping must be confirmed from board docs/schematic before implementation)
- **GPIO Pins:** Use board-specific onboard LED pin assignment; do not repurpose existing I2S/SPI/ADC/relay pins
- **Voltage/Current:** 3.3 V logic, board-integrated LED path expected; verify electrical limits if external LED fallback is used

### State-to-Color Mapping (Proposed)
| SystemState | LED Indication | Behavior |
|---|---|---|
| OFF | Dark / Off | LED completely off |
| STARTING | Blinking (Orange) | 1 Hz blink to signal boot/setup in progress |
| IDLE | Configurable: Dark or Very Dim Green | Default profile favors minimal light emission in idle |
| STREAMING | Solid Blue | Audio stream active; steady blue indicates playback |
| ERROR | Solid Red | State machine in ERROR; user intervention needed |

*All colors are adjustable through a mapping table so users can tune state semantics without touching state-machine logic.*

### Implementation Strategy
1. **New Component:** `LEDComponent` in `lib/led/` with `ILEDListener` interface to observe `SystemState` changes.
2. **Integration:** Register `LEDComponent` as observer in `SystemComponents::setup()`.
3. **GPIO Driver:** Abstraction for RGB PWM control (encapsulate ESP32 PWM/LEDC peripherals).
4. **Blink Pattern:** Soft timer (no FreeRTOS task) for STARTING blink pattern.
5. **No Blocking Delays:** LED setup must not block main boot sequence.
